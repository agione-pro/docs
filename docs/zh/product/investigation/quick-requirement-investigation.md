# AGIOne需求快速调研文档

## 1. 文档目的

本文档用于在客户初次沟通、售前调研、PoC立项或私有化部署评估阶段，快速判断 AGIOne 是否能够满足客户的模型使用需求和算力纳管需求。

调研重点分为两个层面：

| 层面 |需要确认的问题 | 决策影响 |
| --- | --- | --- |
| 模型需求 | 用户需要用什么模型、什么能力、多少人使用、预计调用规模 | 决定 AGIOne 是否已支持该模型，是否需要上架、找资源测试、协议适配或计量方式开发 |
| 算力资源 | 用户是否已有算力或计划采购什么算力，具体 CPU、内存、存储、GPU/NPU 型号等 | 决定客户侧算力资源是否已被 AGIOne适配；未适配时是否需要驱动、镜像、调度或纳管开发 |

## 5. 模型需求与算力资源匹配判断

| 判断维度 |通过条件 | 不通过时的处理动作 |
| --- | --- | --- |
| 模型在支持清单内 | 模型属于已知 on-prem 快速部署模型或已支持协议 | 模型上架、找资源测试、协议适配或计量开发 |
| 模型能力可支持 | Chat、Reasoning、多模态、OCR、Embedding、Reranker、RAG、工具调用等能力已有方案 | 补充模型能力验证或产品开发评估 |
| 算力型号可纳管 | GPU/NPU型号在兼容列表内 | 纳管适配开发或更换算力型号 |
| 算力资源足够 | CPU、内存、显存、卡数、存储、网络满足模型规模 | 调整模型规格、量化方式、并行策略或采购资源 |
| 驱动运行时可用 | OS、Driver、CUDA/CANN、Docker/K8s、镜像版本匹配 | 冻结版本、补充兼容矩阵或现场验证 |
| 性能目标可达 | 并发、RPM、TPM、TTFT、P95/P99满足业务目标 | 压测、扩容、聚合路由、限流或模型替换 |
|计量与治理可落地 | usage、token、调用次数、时长、额度、审计可记录 |计量规则补齐、字段适配或治理流程设计 |

## 6. 初步方案建议

| 场景 | 推荐路径 |说明 |
| --- | --- | --- |
| 模型已支持，算力已适配 |直接进入 PoC 或部署方案设计 | 输出模型、算力、部署、压测和验收计划 |
| 模型已支持，算力未确认 |先做算力环境验证 | 确认 GPU/NPU、OS、驱动、CUDA/CANN、容器、存储和网络 |
| 模型未支持，算力已适配 |先做模型上架和协议/计量评估 | 明确模型来源、镜像、推理引擎、API协议、usage字段和License |
| 模型未支持，算力未适配 | 拆分为模型适配和算力适配两个工作包 | 不建议直接承诺交付周期，需要先完成技术验证 |
| 客户无自有算力 | 基于目标模型和使用规模推荐采购规格 |结合模型参数量、显存、并发、Token规模和高可用要求设计 |
| 客户计划采购国产算力 | 优先确认 Ascend910B/910C、Enflame106、Biren S60 等兼容型号 | 同时确认软件栈、推理引擎和模型适配成熟度 |

## 10.结论模板

### 10.1 可直接支持

客户目标模型属于 AGIOne 已支持或可快速部署范围，客户算力型号位于 AGIOne 纳管兼容列表内，操作系统、驱动、容器运行时、存储和网络条件满足部署前置要求。建议进入 PoC 或部署方案设计阶段。

### 10.2 有条件支持

客户目标模型或算力资源存在部分待验证项，但不存在明确阻塞。建议先完成模型测试、算力环境验证、协议/计量评估或环境整改，再进入正式部署。

### 10.3 暂不支持

客户目标模型不在当前支持范围，且需要新的协议、推理引擎、计量方式或算力纳管适配；或客户算力型号、驱动、OS、存储、网络条件无法满足部署要求。建议进入产品/研发适配评估，不承诺直接交付。
