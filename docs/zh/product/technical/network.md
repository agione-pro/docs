# 网络规划

## 网络架构建议

AGIOne 推荐按管理面、业务面、数据面进行网络分层，确保安全隔离与可运维性。

- 管理面：用于运维管理、监控与日志采集
- 业务面：用于 API 请求接入与业务系统调用
- 数据面：用于模型服务、存储和节点间通信

## 端口策略（示例）

| 类型 | 示例端口 | 说明 |
|------|------|------|
| Web/API | 80 / 443 / 8080 | 对外服务入口 |
| 集群通信 | 10250 / 6443 | Kubernetes 控制与节点通信 |
| 监控日志 | 9090 / 3000 | 指标与可视化（按实际部署） |
| 存储访问 | 9000 / 2049 | 对象存储或网络存储 |

## 安全建议

- 仅开放必要端口，采用白名单策略
- 生产环境优先使用 HTTPS 与内部证书体系
- 跨网段通信需明确路由与防火墙规则
- 核心管理端口建议仅内网可达
