# 其他限制

## 功能边界说明

本页汇总除芯片与版本矩阵外的通用限制条件，帮助项目团队在方案阶段提前识别风险。

## 已知限制项（示例）

### 1) 规模与性能限制

- 超大模型在低显存节点需做量化或分片
- 高并发场景需提前进行容量规划与限流
- 跨地域部署可能影响实时交互时延

### 2) 集成与改造限制

- 部分遗留系统缺少标准 API，需适配开发
- 对接身份系统时需确认协议兼容性
- 非标准网络拓扑可能影响节点发现

### 3) 运维治理限制

- 日志与审计保留周期受存储容量影响
- 极端负载下告警噪声可能增加
- 多租户场景建议设置资源配额

## 风险规避建议

- 在 PoC 阶段完成关键场景压测与演练
- 生产环境预留 20%-30% 资源冗余
- 建立版本升级与回滚机制
