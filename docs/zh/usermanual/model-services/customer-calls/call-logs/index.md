# 调用日志

## 前言

| 项目 | 内容 |
|------|------|
| 适用角色 | User（普通用户） |
| 导航路径 | 客户调用 > 调用日志 |
| 功能定位 | 查看所有客户的模型调用日志，支持按客户、模型、状态等维度筛选 |

## 页面结构

### 搜索区域

页面顶部支持选择时间范围、模型名称、模型类型、调用状态进行筛选。

### 操作按钮区

每个记录提供「详情」按钮。

### 数据列表说明

页面以表格形式按时间倒序展示所有调用记录。

## 操作步骤

### 查看客户调用日志

1. 进入平台首页，点击左侧导航栏的 **"客户调用 > 调用日志"** 菜单，进入客户调用日志页面。
2. 在页面顶部设置筛选参数：
   - 时间范围：选择需要查询的起止日期；
   - 模型：输入模型名称进行模糊搜索；
   - 模型类型：按类型筛选，如对话 / 多模态 / 视频模型等；
   - 调用状态：按状态筛选，如成功 / 失败。
   - 设置完成后，点击 **「搜索」** 按钮加载数据；点击 **「重置」** 可清空所有筛选条件。
3. 查看调用日志列表，表格会按时间倒序展示所有调用记录：
   - 调用时间：请求发生的时间戳；
   - 模型：本次调用的模型名称与标识；
   - 客户名称：发起调用的客户；
   - 模型类型：如对话模型、多模态、视频模型等；
   - 调用状态：成功 / 失败；
   - 使用情况：消耗的输入 / 输出 Token 数（文本 / 多模态模型）或时长（视频模型）；
   - 耗时时间：本次调用的总耗时；
   - 首 TOKEN 时间：从请求发起到返回第一个 Token 的时间，用于评估响应速度；
   - 失败类型：如系统异常、限流等（仅失败记录显示）；
   - 错误信息：失败的具体原因描述（仅失败记录显示）；
   - 操作：点击「详情」可查看单条调用的完整信息。

### 查看单条调用详情

1. 点击任意记录的 **「详情」** 按钮，打开详情弹窗。
2. 查看完整的调用信息：
   - 基本信息：模型名称、调用时间、模型类型、总耗时、首 Token 时间、调用状态；
   - 消耗情况：输入 / 输出 Token 消耗明细。

#### 参数说明

| 字段名称 | 字段类型 | 示例 | 说明 |
|----------|----------|------|------|
| 调用状态 | 标签 | `成功 / 失败` | 本次调用的结果，分为「成功」和「失败」 |
| 使用情况 | 文本 | `输入: 22 Tokens / 输出: 15 Tokens` | 本次调用消耗的资源：文本 / 多模态模型为输入和输出 Token 数，视频模型为生成时长 |
| 耗时时间 | 数值 | `1417 ms` | 从请求发起到完全结束的总时间，反映模型的整体性能 |
| 首 TOKEN 时间 | 数值 | `320 ms` | 从请求发起到返回第一个 Token 的时间，反映模型的响应速度 |
| 失败类型 | 标签 | `系统异常 / 限流` | 失败请求的分类，如系统异常、限流触发、模型错误等 |
| 错误信息 | 文本 | `Rate limit exceeded` | 失败请求的具体错误描述，可用于问题排查 |

## 注意事项

* 可通过顶部的时间范围、模型名称、模型类型或调用状态进行组合筛选，快速定位目标记录。
* 点击任意记录的「详情」按钮可查看该次调用的完整信息。
