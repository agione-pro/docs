# AGIOne 部署配置要求文档

> 本文档明确 AGIOne 在生产环境（Production）与 PoC 环境的部署前置条件，涵盖硬件规格、软件依赖、网络配置与账号权限等基础要求。实际交付前应结合交付包兼容矩阵、驱动版本和现场评估结果进行最终确认。

## 2. 部署形态概述

| 部署形态             | 支持状态 | 典型场景                                  |
| ---------------- | ---- | ------------------------------------- |
| All-in-One 单机    | 支持   | 演示、测试、PoC、小规模部署                       |
| 本地 Host-mode 多节点 | 支持   | 至少 4 台机器：2 应用节点 + 1 中间件节点 + 1 数据库备库节点 |
| Kubernetes / 云原生 | 支持   | 已有 K8s 集群或 AGIOne 交付 kube-cluster     |
| 私有化部署            | 支持   | 企业内网、IDC、私有云、专有云                      |
| 离线或弱网            | 支持   | 无公网或受限公网环境                            |

> **说明**：单机部署不适合承载高可用生产负载；生产环境推荐多管理节点高可用架构。

## 3. 硬件配置要求

### 3.1 管理节点（Management Nodes）

承载 AGIOne 核心服务、数据库、中间件与 K8s 管理插件。

| 工作节点规模 | K8s 资源要求 | AGIOne 服务资源要求 | 最小节点数 |
| --- | --- | --- | --- |
| 1–5 节点 | 1C 4GiB | 7C 12GiB | 3 |
| 6–10 节点 | 2C 8GiB | 7C 12GiB | 3 |
| 11–100 节点 | 4C 16GiB | 12C 24GiB | 3 |
| 101–250 节点 | 8C 32GiB | 12C 24GiB | 3 |
| 251–500 节点 | 16C 64GiB | 24C 48GiB | 3 |
| 500+ 节点 | 32C 128GiB | 32C 64GiB | 3 |

> - 管理节点**不要求**配置 XPU（GPU/NPU/TPU）。
> - 生产环境推荐 **≥3 节点高可用部署**；单节点替代方案（≥16C / ≥32GiB）仅适用于测试/PoC。
> - 共享存储（Block Storage）**推荐配置**，系统盘建议 ≥1024 GiB。
> - 建议工作节点总量控制在 **1000 以内**。

### 3.2 XPU / NPU 算力节点（Worker Nodes）

承载推理实例、IDE、K8s 工作节点服务（kubelet、proxy 等）。

| 节点类型 | CPU（预留） | 内存（预留） | 系统盘 | 数据盘 | 节点数 |
| --- | --- | --- | --- | --- | --- |
| XPU 资源节点 | ≥16 核 | ≥32 GiB | ≥100 GiB | ≥2048 GiB | ≥1 |
| Ascend NPU 节点 | ≥8 核（预留） | ≥16 GiB（预留） | ≥200 GiB | — | ≥1 |

> - 算力节点**至少需要 1 张 XPU**（GPU、NPU、TPU 等）。
> - 同一 Kubernetes 集群建议管理**相同 CPU 架构**，避免异构架构混入导致调度不稳定。
> - Ascend NPU 节点开放端口：`6443`、`8090`、`32761`–`32765/TCP`。

### 3.3 共享存储

| 类型 | 系统盘 | 节点数要求 |
| --- | --- | --- |
| 共享存储（NAS/IPFS 等） | ≥2048 GiB | ≥2（推荐） |

> 共享存储用于模型权重、镜像、日志、MinIO、数据库和中间件持久化存储。管理节点与工作节点可共用或分离部署。

## 4. 软件配置要求

### 4.1 操作系统支持

| 操作系统 | 支持状态 | 说明 |
| --- | --- | --- |
| Ubuntu 22.04 | **推荐** | 生产首选 |
| Ubuntu 20.04 | 可选 | 作为推荐范围 |
| CentOS / Rocky 7.x / 8.x | 条件支持 | 需现场验证 |
| Kylin / UOS | 需评估 | 需结合交付包确认 |
| 本地单机最低规格 | 8 核 / 16 GiB / 200 GiB 可用磁盘 | 仅限 All-in-One / PoC |

### 4.2 基础软件依赖

| 组件 | 推荐版本 | 说明 |
| --- | --- | --- |
| Docker | Docker 24.x+ | 需匹配操作系统、内核、镜像包和离线安装方式 |
| Docker Compose | - | 由 AGIOne 安装器自动安装或修复 |
| Kubernetes | 1.24–1.29 | 需结合交付包、GPU/NPU 插件、CNI、CSI、Ingress Controller 验证 |
| Helm | Helm 3.10+ | 仅适用于 Helm / Chart 交付形态 |
| Python | 由 bundle 内置离线运行时提供 | 目标机无需预装 python3 |

### 4.3 中间件组件

| 组件 | 默认端口 | 说明 |
| --- | --- | --- |
| MariaDB | 3306 | 数据库持久化存储、备份恢复需部署前确认 |
| Redis | 6379 | 缓存与会话，密码一致性需部署前确认 |
| Nacos | 8848 / 8849 | 配置中心，服务注册发现 |
| Kafka | 9092 | 消息队列 |
| MinIO | 9000 / 9001 | 对象存储（日志、模型产物等） |
| OpenResty / Nginx | 80 / 443 | 入口代理、作业访问代理 |

## 5. 网络配置要求

### 5.1 网络分层

| 网络类型 | 要求 | 说明 |
| --- | --- | --- |
| 内部网络 | **必须** | 管理节点与 XPU 算力节点通过内部局域网连接 |
| 外部互联网 | **推荐** | 可访问外部网络（适用于模型下载、License 校验等场景） |
| RDMA 网络 | **可选** | 单卡推理可不强依赖；多机多卡推理或训练建议 100Gbps 以上 RDMA |

### 5.2 关键端口

| 端口 / 协议 | 用途 | 所属节点 |
| --- | --- | --- |
| `18090/TCP` | Console Web 入口（默认） | 管理节点 |
| `443/HTTPS` | 外部 HTTPS 入口（可选） | 管理节点 |
| `80/TCP` | Nginx 入口 | 管理节点 |
| `8089/TCP` | 作业访问代理入口 | 管理节点 |
| `3306/TCP` | MariaDB | 管理节点 |
| `6379/TCP` | Redis | 管理节点 |
| `8848/8849/TCP` | Nacos | 管理节点 |
| `9000/9001/TCP` | MinIO API / Console | 管理节点 |
| `9092/TCP` | Kafka | 管理节点 |
| `6443/TCP` | Kubernetes API Server | Ascend NPU 节点 |
| `8090/TCP` | 服务端口 | Ascend NPU 节点 |
| `32761`–`32765/TCP` | 算力节点通信端口 | Ascend NPU 节点 |

> **注意**：文档中存在 `443`、`18090`、`80`、`8089` 等多种端口表达，区分外部 HTTPS 入口、门户/API 入口、Nginx 入口和作业访问代理入口；正式部署前应形成统一端口规划表。

### 5.3 网络要求清单

| 要求项 | 说明 |
| --- | --- |
| 节点间内网互通 | 管理节点与算力节点之间通过本地 LAN 连接 |
| SSH 可达 | 部署操作需 SSH 登录目标主机 |
| 时间同步 | 多节点部署要求 NTP 或等效时间同步，建议时钟偏差控制在 1 秒以内 |
| 域名解析 | Ingress / DNS / TLS 依赖域名解析，外部访问和证书验证需上线前确认 |
| 镜像仓库可达 | 云原生部署要求所有节点可访问镜像仓库，提前验证 registry、认证 Secret 和 image pull |
| 防火墙规则 | 确认上述端口在防火墙或安全组中已放行 |

## 6. 账号与权限要求

### 6.1 操作系统账号

| 要求 | 说明 |
| --- | --- |
| 执行用户 | 推荐 `root` 或等效权限用户，避免 Docker、目录权限和系统服务权限问题 |
| SSH 权限 | 部署操作需具备目标主机 root 或 sudo 权限 |
| 目录写权限 | 对 `/opt/hyperone` 所在分区有写权限（建议 200 GiB 可用空间以上） |

### 6.2 AGIOne 系统账号体系

| 角色             | 业务身份   | 说明         |
| -------------- | ------ | ---------- |
| `operator`     | 运营方管理员 | 全平台唯一运营方角色 |
| `normal`       | 普通用户   | 常规业务操作     |


### 6.3 Kubernetes 权限（云原生部署）

| 要求 | 说明 |
| --- | --- |
| kubeconfig 可用 | 具备目标集群的操作凭证 |
| 命名空间权限 | 命名空间读写权限充足 |
| 节点 Ready | 所有目标节点状态为 Ready |
| StorageClass | RWO 持久卷（数据库、中间件）与 RWX 共享存储（模型、日志）就绪 |
| Ingress Controller | 入口控制器可用，域名和证书就绪 |

## 7. 存储配置要求

| 存储类型 | 用途 | 建议规格 |
| --- | --- | --- |
| 块存储（管理节点） | 数据库、MinIO 持久化 | ≥400 GiB |
| 共享存储（NAS/IPFS） | 模型权重、镜像、日志 | ≥2048 GiB |
| 本地磁盘（All-in-One） | `/opt/hyperone` 运行目录 | ≥200 GiB（检测容差约 160 GiB 以上可通过） |

> - 生产环境建议预留 **20%–30% 资源冗余**。
> - 云原生部署需提前验证 PVC 创建、挂载和备份恢复。
> - 数据库主备、备份机制和回滚计划需在部署前确认。

## 8. PoC 环境快速配置参考

适用于演示、测试、PoC 等小规模场景（All-in-One 单机部署）。

| 项目 | 推荐值 | 说明 |
| --- | --- | --- |
| 操作系统 | Ubuntu 22.04 LTS | Linux |
| CPU | 8 核 | 检测允许少量保留损耗，约 7.6 核以上可通过 |
| 内存 | 16 GiB | 检测允许少量系统保留损耗，约 15.2 GiB 以上可通过 |
| 可用磁盘 | 200 GiB | `/opt/hyperone` 所在分区，检测允许约 20% 文件系统保留损耗，约 160 GiB 以上可通过 |
| 执行用户 | `root` | 推荐 root 安装 |
| 安装方式 | `./agione quick` | 一键完成解包、检查、配置、镜像加载与服务启动 |
| 访问入口 | `http://<host>:18090/modelone/` | 由安装结果输出为准 |

### PoC 快速检查清单

```bash
# 1. 上传并解压 bundle
scp agione-bundle-set-<version>.tar.gz root@<target>:/opt/hyperone/
cd /opt/hyperone && tar -zxvf agione-bundle-set-<version>.tar.gz

# 2. 一键安装
chmod +x ./agione
./agione quick

# 3. 安装后检查
./agione health
./agione ps

# 4. 浏览器访问
# http://<target-ip>:18090/modelone/

# 5. 导出交付包（推荐）
./agione handover
```
