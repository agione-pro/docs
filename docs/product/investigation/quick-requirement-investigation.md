# Quick Requirements Investigation

主要用于在前期对用户现有的 AI 场景需求进行调研。信息收集完成后，可进一步评估需求满足性并制定落地方案，推进 AGIOne 产品实施。

## 调研目标

- 明确客户业务目标与成功指标
- 识别优先落地场景与可行性
- 形成可执行的 PoC 或上线计划

## 快速调研清单（示例）

### 1) 业务背景

- 当前业务痛点是什么？
- 期望通过 AI 解决哪些核心问题？
- 项目涉及哪些部门和角色？

### 2) 场景需求

- 目标场景：知识问答 / 文档处理 / 智能助手 / 其他
- 日均请求量与峰值并发预估
- 对响应时延、准确率、稳定性的要求

### 3) 数据与安全

- 数据来源与数据类型（文本、图片、结构化数据）
- 是否涉及敏感数据或合规要求
- 数据是否允许出域、是否要求本地化部署

### 4) 集成需求

- 需对接的系统（OA、CRM、客服、门户等）
- 接口方式（API、消息队列、文件交换）
- 单点登录与权限体系要求

## 调研输出物（示例）

| 输出项 | 内容 |
|------|------|
| 场景优先级清单 | 按业务价值和实施难度分级 |
| 需求满足性评估 | AGIOne 能力与需求映射结果 |
| 实施建议 | PoC 范围、周期、资源建议 |
| 风险清单 | 技术/数据/组织协同风险 |
